인하 대학교 (Inha University)

처음 입학할 때의 전공은 정보공학계열이었고, 2학년이 될 때 컴퓨터정보공학을 선택하였습니다. 고등학교의 프로그래밍 실무란 수업에서 처음 코딩을 하였는데, 너무 재미있었고 그 때는 보안전문가가 되고 싶어서 인하대학교 정보공학계열로 진학하였습니다. 대학교 1학년은 암울했습니다. 고등학교에서 수학과 물리학에 대해 깊은 공부를 하지 않았었는데, 대학교의 기초 교양과목을 수강하는 것이 너무 힘들었습니다. 하지만 전공 과목은 정말 좋았습니다. 1학년 1,2학기에 걸쳐 객체지향프로그래밍을 수강하였는데, 정말 재미있었습니다. 2학년 1학기에는 프로그래밍 동아리의 Java 스터디 모임을 나가게 됩니다. 친구, 동생들과 함께 미리 공부해 온 내용을 공유하고 발표하고 같이 프로그램도 만들면서 Java 언어를 처음 접했습니다. (이 때가 2011년 이니까 약 2년전 이네요. 그러고 보니 저의 Java 경력은 2년 조금 넘는군요! 더 열심히 해야겠습니다.) 슬슬 경험도 쌓이고 자신감도 생기면서 프로젝트를 계획하게 됩니다.

Java 스터디를 함께 했던 친구들과 "인하대 시간표 작성 & 강의 후기 사이트"를 만들기로 하면서 2011년의 여름방학을 오로지 이 프로젝트를 위해 불 태우게 됩니다. 시간표 작성은 경우의 수를 계산하는건데, 사용자가 원하는 과목을 여러개 선택하면 그 과목을 이용하여 최적의 시간표를 만들어 주는 기능입니다. 네, 다른 학교에서는 이용 가능한 서비스입니다. 하지만 인하대!에는 이 서비스가 없었고, 만들어보고 싶었습니다. (미리 말씀드리면, 이 프로젝트는 중단 되었습니다.) 일단 시간표 데이터베이스가 필요했는데, 참 무식하게 했습니다. 인하대 수강신청 홈페이지에서 전공별 과목을 하나 하나 copy & paste 하여 엑셀로 옮기고, 그 엑셀 시트를 CSV 파일로 export 하여 MySQL 데이터베이스에 import 하고... 우리 학교의 전공이 그렇게 많은지 그 때 제대로 알았습니다.

아무튼 시간표 작성 서비스는 끝내 완성하지 못했습니다. 기술적으로 한계에 부딪혔고 2학기가 개강하였죠. (졸업하기 전에 제대로 만들어서 오픈할 겁니다. 학부 졸업일 지 혹은 석사 졸업일 지는 모릅니다.ㅋ) 그런데 강의 후기 사이트는 어느정도 완성을 하였습니다. 교내 자체에도 강의를 평가하는 시스템이 있지만 그 데이터를 학생들이 열람할 수는 없죠. (총 평점만 보이는 수준입니다.) 만든 강의 후기 사이트를 웹 호스팅 서버에 올리고, 도메인 네임 연결하고...(http://inha-dic.com) 인하광장에 서비스 홍보 글을 올리게 됩니다. 사실 올릴까 말까 한참 고민했는데, 이유는 보안 문제였습니다. 간단히 그냥 Tomcat 서버에서 JSP 페이지 만으로 이루어진 웹 어플리케이션인데 만들면서 보안에 많은 신경을 쓰지 못(안)했습니다. 고민 끝에 결국 인하광장(교내 커뮤니티)에 홍보 글을 올렸는데 제법 많은 학우들이 관심을 가져 주었고, 추천도 좀 받았습니다. 하지만 글을 올리고 한 시간? 두 시간 만에 내리게 됩니다. 도저히 보안 문제에 대해 자신이 없었고, 아직은 때가 아니다!

그러고 보니 2학년 2학기에 정말 중요한 일이 있었죠. 연구실에 들어가게 됩니다. 이 학기에 자바프로그래밍 강의를 수강하면서 Java 언어에 대해 제대로 배우고 그 매력에 푹 빠지게 됩니다. 어느날 인하광장을 보는데 지능기술연구실에서 학부 인턴을 모집하고 있었습니다. 웹 프로그래밍 가능한 학생을 원했는데, 딱 "아! 나를 찾는구나" 싶었습니다. 바로 지원했고 합격하고 들어가게 됩니다. 2학년이 되면서 부터 대학원 진학을 원했고 좋은 기회라고 생각했기 때문에 망설일 것이 없었습니다. 공부하고 작업할 내 책상도 있고 PC도 있고 최고의 환경이 만들어 집니다. 겨울 방학이 시작되면서 본격적으로 연구실 생활을 하게 됩니다. 처음 진행했던 프로젝트는 사진 추천 시스템이었습니다. 사용자 간의 유사도를 분석해서 좋아할 만한 사진을 추천해주는 서비스입니다. 여기에는 협력적 필터링(collaborative filtering) 기법이 들어갑니다. 기존의 무비렌즈(Movie lens) 데이터를 이용한 프로그램이 연구실에 있었고 그 소스를 기본으로 하여 프로젝트를 진행했습니다. 한 달여 만에 프로토타입 수준의 결과물이 나왔고, 그 이후로는 기능 추가 및 보완을 진행했습니다. 함께한 조교님은 연구 논문을 쓰게 됩니다. 논문은 2012년 6월쯤에 발표했는데, 저도! 부저자로 이름을 올리게 됩니다. 정말 좋았고 뿌듯했습니다. 내가 가장 좋아하는 것으로 뭔가를 만들어서 어떤 결과를 얻게 된 것이었습니다. 그 이후로 인터십하러 비행기 타는 날, 이틀전 까지 연구실에 머물렀죠.

3학년 1학기가 시작되기전에... 겨울 방학에는 또 게임 개발 스터디에 참여하게 됩니다. 이매진컵 진출을 목표로 진행된 스터디인데, 결과물은 괜찮았습니다. 아이디어도 좋았고 최종적으로 세계 예선 1,2차 통과하고, 그러나 중간에 그만두게 됩니다. 개인적으로 너무 힘들었거든요. 연구실 생활, 학교 생활, 게임 스터디 까지 감당하기에는 벅찼습니다. 사생활이 없었죠. 방학때는 괜찮았는데, 개강을 하니 정말로 남는 시간이 없더라구요. 아무튼 그렇게 게임스터디를 끝내고 난 후에는 또 다른 스터디에 참여하게 됩니다. 연구실 교수님께서 연구실 차원에서 스터디를 모집하게 됩니다. Google 입사를 목표로하여 2년 이상의 기간 동안 기초 영어, 클라우드 컴퓨팅, 빅데이터 등에 대해 학습하기로 계획된 스터디였습니다. (스터디 이름도 GoGoogle 입니다. 패기 넘치죠.) 강의 시간에 교수님께서 스터디를 홍보하시고 학생들에게 관심 있으면 찾아오라고 말씀하셨는데, 저는 그 당시에는 망설였습니다. 게임 스터디는 벅차서 그만뒀는데, 이것 역시 힘들겠다 싶었습니다.

지원하지 않았습니다. 그 스터디가 1주 혹은 2주 진행 되었을 무렵, 교수님께서 제게도 참여를 권유하시게 됩니다. 저도 사실 Google 이라는 회사에 대해 동경이 있었고, 교수님께서 권유하시니 한 번 참여해 보자 생각이 들게 됩니다. 그래서 저는 뒤늦게 참여하게 되었습니다. 학기중에는 매주 토요일에 스터디를 하게 됩니다. 하이테크센터 12층 세미나실을 빌려 아침 9시에 시작했는데, 영어로 진행되었습니다. 발표도 질문도 말입니다. 주제는 위에서 언급했듯이 클라우드 컴퓨팅, 빅 데이터입니다. 스터디 초반에는 클라우드 컴퓨팅의 기초 개념에 대해 학습하였습니다. 그리고 여름방학이 시작되면서 서서히 진짜 기술에 대해 공부하기 시작했죠. Hadoop, Cassandra, HBase, Amazon AWS 같은 것 말입니다. 테스트 환경을 만들어 프로그램도 작성해보고 서로 공유하면서 발전해 나갔습니다. 빅 데이터 쪽은 환경 세팅하는 것이 절반은 하는 것 같습니다. 처음할 때는 뭐가 그렇게 잘 안되는지. 많은 시행착오(삽질)를 겪었죠. 연구실에서 밤새 작업하고 다음날 스터디 발표하고. 정말 재미있었습니다. 시간 정말 잘가더라구요. 스터디 발표는 매주 토요일 같은 시간에 하되, 평일에 멤버끼리 자주 모였죠. 팀 프로젝트를 진행했거든요. 물론 그 전에 개인 프로젝트도 하나씩 만들어 발표하였습니다. 지금까지 학습한 빅데이터 기술을 이용해 뭔가를 만들어야 하는데, 많은 아이디어가 오고 간 끝에 결정하게 됩니다.

내용 기반 비디오 검색 시스템(Content based video retrieval). 쉽게 말하면 사진을 가지고 영상을 찾는겁니다. 유튜브 같은 서비스 이용할 때는 보통 문자열을 입력해 영상을 검색한다면, 이 서비스는 그림을 이용해 영상을 찾습니다. (예를 들어, 사용자가 어떤 영화의 한 장면의 이미지를 파일로 가지고 있습니다. 그런데 만약 사용자가 그 영화가 어떤 영화인지 알고 싶다면 어떻게 해야 할까요? 네이버 지식인 같은 곳에 올려 질문할 수도 있겠지만 이 서비스를 이용한다면 해당 장면이 포함된 영상을 찾아내어서 '그건 달콤한 인생의 한 장면이에요!' 라고 알려주는거죠.) 일단 유사 기술에 대해 조사를 시작했습니다. 나름대로 많이 찾아봤는데, 유사한 서비스는 국내에 단 한 곳 있더라구요. 지금은 '이미디오' 라는 이름으로 알려진 서비스에요. 이미디오의 기술은 상당했습니다. 서비스를 크롬 브라우저에 확장 기능 형태로 제공하기 때문에 편리성도 우수하고요. 인덱싱 된 비디오 숫자도 어마어마 했죠. 그래도 우리는 만들기로 합니다. 아이디어가 나오고 일주일 이내에 프로토타입 수준의 결과가 나왔습니다. Hadoop, Lucene, OpenCV 등의 기술을 사용해서 말이에요. 토요일 세미나 시간에 교수님 앞에서 진행한 것을 보여드렸는데, 글쎄 엄청 혼났어요. Research를 많이 안했다구요.
